# 小鈴 C01基本移動シート 歩行GIF失敗評価

## 評価対象

- マネキン一覧: `03_named_images/小鈴/01_mannequin_body/小鈴_マネキン動作一覧_C01基本移動8種_01.png`
- 小鈴化シート: `03_named_images/小鈴/08_generated_scenes/01_歩行/小鈴_C01基本移動8種_キャラクター化_01.png`
- GIF: `03_named_images/小鈴/08_generated_scenes/01_歩行/小鈴_C01基本移動_02-03-04-08.gif`
- GIF使用順: `02 -> 03 -> 04 -> 08`

## 結論

このGIFは、シームレスな歩行モーションとして不採用。

最大の原因は画像生成の描画精度ではなく、異なる8種類の移動を並べた「動作一覧シート」を、一つの歩行を時系列に分解した「連続動作シート」として扱った工程判断にある。

`C01 基本移動一覧` はポーズ辞書であり、GIFフレームとして使用してはいけない。

## 構図ごとの評価

|番号|本来の役割|単体評価|前後の連続性|
|---|---|---|---|
|01|自然立ち・待機|正面の待機姿勢として成立|側面歩行へ直接つながらない|
|02|通常歩行|右向き側面の接地姿勢として概ね成立|03とは速度と歩幅が変わるだけで、次の歩行位相になっていない|
|03|早歩き|早歩きの単発姿勢として成立|02と同じ脚配置に近く、左右脚の交代がない|
|04|中速走行|走行・滞空姿勢として成立|歩行から突然走行へ変わり、接地が消える|
|05|急停止|減速姿勢として成立|カメラが斜め前へ変わるため歩行GIFには使えない|
|06|方向転換|回転姿勢として成立|背面へカメラ方向と身体方向が変わる|
|07|横移動|正面のサイドステップとして成立|進行方向とカメラが歩行フレームと異なる|
|08|後退|右向き側面の後退姿勢として概ね成立|04の前進走行から後退へ突然反転する|

## シームレス歩行にならなかった理由

### 1. 一歩分の位相が存在しない

歩行ループには、最低でも以下の左右対称な位相が必要。

```text
01 左足接地
02 左脚荷重・沈み込み
03 右足通過
04 左脚蹴り出し・持ち上がり
05 右足接地
06 右脚荷重・沈み込み
07 左足通過
08 右脚蹴り出し・持ち上がり
```

C01には、通常歩行、早歩き、走行、停止、方向転換、横移動、後退が混在している。左右の接地が交互に現れないため、一歩分の周期が成立しない。

### 2. 02と03は別位相ではなく速度違い

02と03は、同じ方向へ似た脚配置で進む単発ポーズ。歩幅と腕振りは変わるが、接地から荷重、通過、蹴り出しへ進んでいない。

### 3. 04で歩行から走行へ変わる

04は膝を高く上げた走行・滞空姿勢。歩行には両足が同時に浮く滞空期がないため、02、03からつなぐと速度と運動種別が突然変わる。

### 4. 08は後退

08は前進歩行の次フレームではなく、後方へ足を置く別動作。04から08へつなぐと、前進走行から後退へ切り替わって見える。

### 5. カメラと身体方向が統一されていない

C01全体では正面、側面、斜め前、斜め後ろが混在する。一つのGIFではカメラを完全側面など一方向に固定する必要がある。

## フレーム位置の測定結果

小鈴化シートを4列×2段で分割し、白背景から人物領域を測定した。

GIFに使用したフレーム:

|番号|人物サイズ|中心X|人物下端Y|
|---|---:|---:|---:|
|02|203×331px|202.5|372|
|03|218×326px|204.0|367|
|04|191×334px|194.5|372|
|08|187×330px|193.5|354|

問題:

- 02と08で足元の下端が18px違う。
- 中心Xが最大10.5pxずれる。
- 人物幅が187pxから218pxまで変化する。
- 同じパネルサイズで単純に切り出しただけでは、GIF内で人物が上下左右へ揺れる。

これは生成側のフレーム配置の揺れと、切り出し後に足元・骨盤基準で整列しなかった実装の両方が原因。

## 成功していた点

- 小鈴の黒髪ショートボブ、白黒武術服、黒レギンス、白黒靴は概ね維持された。
- 純白背景とパネル分割は、切り抜き用途に適している。
- 各パネルは単発の移動ポーズ辞書として利用できる。
- 番号除去と4×2シートからの自動切り出しは正常に動作した。

## 原因の切り分け

### 主因: 資料運用と意図理解

- 資料には、C01が「異なる8動作の一覧」であり、GIFには使わないと書かれていた。
- それにもかかわらず、キャラクター化とGIF作成時にC01を歩行連番として扱った。
- 「基本移動8種」という名前が「歩行8フレーム」と混同しやすかった。
- GIF化前に、フレームが同じ動作・同じ方向・時系列かを確認する工程がなかった。

### 副因: 画像生成の限界

- 一枚のシート内でも人物サイズ、足元、骨盤位置が少しずれる。
- 衣装の前垂れ、腕、脚の形がフレーム間で完全には固定されない。
- 画像生成モデルは、ポーズ一覧は得意でも、厳密なアニメーション中割りの一貫性は保証しない。

画像生成側だけを修正しても、元資料が非連続なら歩行ループにはならない。

## 対策

### 1. IDと名称を分離する

- `CAT-MOVE-01`: 基本移動カタログ。異なる8動作。GIF禁止。
- `SEQ-WALK-01`: 歩行8フレーム。GIF可。
- `SEQ-RUN-01`: 走行8フレーム。GIF可。

今後、`一覧`、`カタログ`、`CAT` を含む画像はGIF入力にしない。

### 2. 歩行はL01専用プロンプトで作り直す

`02_prompts/小鈴_マネキンアクションシート_基本動作プロンプト.md` の `L01 歩く` を使用する。

- 完全な側面カメラで固定。
- 全8フレームで画面右向き。
- 左右の接地、荷重、通過、持ち上がりを交互に配置。
- 08から01へ自然につなげる。
- 走行、停止、方向転換、後退を混ぜない。

### 3. その場歩きとして生成する

GIF確認では人物の骨盤中心Xを固定する。

- 人物は各パネル中央に固定。
- 骨盤中心のX座標を全フレームで同じにする。
- 足元の基準線を全フレームで同じY座標にする。
- 前進距離は描かず、脚と腕の周期だけを表現する。

### 4. マネキン段階でGIF審査する

```text
L01マネキン生成
-> 8パネル切り出し
-> 足元・骨盤基準で整列
-> マネキンGIF作成
-> シームレス判定
-> 合格した場合だけ小鈴へ変換
```

キャラクター化を先に行わない。

### 5. GIF化前の必須ゲート

- [ ] 一つの動作だけか。
- [ ] カメラ方向が全フレーム同じか。
- [ ] 進行方向が全フレーム同じか。
- [ ] 左右脚が交互に接地しているか。
- [ ] 接地、荷重、通過、持ち上がりがあるか。
- [ ] 最終フレームから先頭フレームへつながるか。
- [ ] 人物の骨盤中心と足元基準線が揃っているか。
- [ ] `CAT` または `一覧` シートではないか。

一つでも満たさない場合はGIFを作らない。

### 6. 再生速度

今回の260ms/コマは約3.8fpsで、歩行としてはかなり遅く見える。

8フレーム歩行では、まず100から140ms/コマを基準に確認する。動作検証用に遅く見る場合だけ180から240msへ上げる。
