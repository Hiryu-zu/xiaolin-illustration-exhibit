# ChatGPT 画像生成チャット内マーカー運用ルール

このチャットでは、あとから画像・プロンプト・note記事素材を整理しやすくするために、重要な判断や完成プロンプトに必ずマーカーを付けて記録してください。

目的は、後日Codexや別のAIがチャット全文を読まなくても、必要な情報だけを短時間で抽出できるようにすることです。

## 基本方針

- 画像生成のたびに、よかった点・悪かった点・完成版プロンプトを短く構造化して残す
- 長い会話文ではなく、あとから機械的に拾えるマーカーを使う
- note記事化に使える観点も同時に残す
- 汎用プロンプト化できる要素と、固定キャラ専用の要素を分けて書く

## 使用するマーカー

優先して使うマーカーは次の3つです。

```text
[GOOD]
[BAD]
[FINISHED_PROMPT]
```

必要に応じて、補助マーカーとして次も使ってください。

```text
[KEEP]
[FIX]
[NOTE]
[RULE]
```

## [GOOD] 採用候補の記録

生成結果がよかった場合、次の形式で記録してください。

```text
[GOOD]
タイトル:
画像:
用途:
よかった点:
残す要素:
汎用化できる要素:
固定キャラ依存の要素:
noteで語る見どころ:
次に試すなら:
```

### 記入例

```text
[GOOD]
タイトル: 夕日の武術構え
画像: 1枚目
用途: note掲載候補、戦闘構図紹介
よかった点: 逆光、低い構え、道場の奥行きがよい
残す要素: 夕日、木造道場、片手を前に出す構え、静かな緊張感
汎用化できる要素: 武術家、夕日の道場、低い防御姿勢、間合いを測る空気
固定キャラ依存の要素: 小鈴の黒髪ショートボブ、白黒中華武術衣装、琥珀色の瞳
noteで語る見どころ: 動き出す直前の圧を、派手な攻撃ではなく構えと光で見せている
次に試すなら: 手の形をさらに安定させる、足元の接地感を強める
```

## [BAD] 不採用・改善点の記録

失敗や不採用も、後でネガティブ指定や改善ルールに使うために記録してください。

```text
[BAD]
タイトル:
画像:
不採用理由:
残してよい要素:
直す要素:
次回の注意:
ネガティブ指定候補:
```

### 記入例

```text
[BAD]
タイトル: 跳び蹴り構図
画像: 2枚目
不採用理由: 手指が破綻、顔が幼く見える、脚の関節が不自然
残してよい要素: 夕日の逆光、石畳、動きの方向
直す要素: 手指、年齢感、脚の関節、衣装の黒白比率
次回の注意: 動きは強くしてよいが、人体の接地感と関節を優先する
ネガティブ指定候補: broken fingers, incorrect limb joints, childish proportions
```

## [FINISHED_PROMPT] 完成プロンプトの記録

プロンプトが完成したら、必ずこの形式で記録してください。

```text
[FINISHED_PROMPT]
タイトル:
用途:
対象画像:
汎用化:
固定キャラ:
優先順位:
共通ルール:
画像固有:
固定キャラ差分:
ネガティブ:
プロンプト:
```

### 記入例

```text
[FINISHED_PROMPT]
タイトル: 夕日の武術構え
用途: note記事、参考画像プロンプト、戦闘構図紹介
対象画像: xiaolin_009_1a7a89db.png
汎用化: 可能
固定キャラ: 小鈴（シャオリン）
優先順位: 構図 > 光 > ポーズ > 人体の自然さ > 衣装 > 表情
共通ルール: 高品質アニメ調、自然な人体、手指破綻を避ける、文字やロゴを入れない
画像固有: 夕日の木造道場、低い防御姿勢、片手を前に伸ばす、静かな緊張感
固定キャラ差分: 黒髪ショートボブ、琥珀色の瞳、白黒中華武術衣装、しなやかな武術体型
ネガティブ: extra characters, weapons, text, logos, distorted hands, broken fingers, incorrect limb joints
プロンプト:
Use the reference image as the main composition and mood guide...
```

## 補助マーカー

### [KEEP]

残したい要素を短く列挙します。

```text
[KEEP]
夕日の逆光
低い防御姿勢
木造道場
石畳の長い影
静かな緊張感
```

### [FIX]

次回直したい要素を短く列挙します。

```text
[FIX]
手指の破綻
顔が幼くなる問題
足元の接地感
衣装の黒白比率
```

### [NOTE]

note記事で語る見どころや文章化の観点を残します。

```text
[NOTE]
この構図は、攻撃の瞬間ではなく「動き出す直前の圧」を見せるためのもの。
夕日の逆光で輪郭を強調し、武術家としての集中力を表現している。
```

### [RULE]

今後も使い回す共通ルールを残します。

```text
[RULE]
戦闘イラストでは、派手なエフェクトよりも人体の自然さ、重心、接地感を優先する。
小鈴の場合、細すぎず筋肉質すぎない、しなやかな武術体型を維持する。
```

## 読み取り優先順位

後でCodexや別AIが読むときは、次の順番で参照します。

```text
[FINISHED_PROMPT]
↓
[GOOD]
↓
[KEEP] / [FIX] / [NOTE]
↓
[BAD]
↓
[RULE]
```

`[FINISHED_PROMPT]` がある場合は、それを最優先します。  
`[FINISHED_PROMPT]` がない場合は、`[GOOD]` と `[KEEP]` をもとに構造データを作ります。  
`[BAD]` は失敗理由やネガティブ指定の材料として使います。

## 汎用化と固定キャラの分離ルール

必ず次の2つを分けて考えてください。

### 汎用化できる要素

- 場所
- 構図
- ポーズ
- 光
- 雰囲気
- カメラ視点
- 背景ディテール
- ネガティブ指定

### 固定キャラ依存の要素

- キャラ名
- 髪型
- 目の色
- 衣装
- 年齢
- 体型
- 性格
- 表情の傾向

## 画像生成後の短い運用テンプレ

毎回、画像生成後に最低限これだけ記録してください。

```text
[GOOD] または [BAD]
タイトル:
画像:
理由:
残す要素:
直す要素:
汎用化できる要素:
固定キャラ依存の要素:
noteで語る見どころ:
```

完成プロンプトまで作った場合は、最後に必ず `[FINISHED_PROMPT]` を追加してください。

## AIへの依頼文テンプレ

このルールを読んだAIは、画像生成やプロンプト整理のあとに、次のように出力してください。

```text
以後、このチャットでは画像生成結果を整理するため、[GOOD] [BAD] [FINISHED_PROMPT] のマーカーを使ってください。
採用候補は [GOOD]、不採用や改善点は [BAD]、完成プロンプトは [FINISHED_PROMPT] として、汎用化できる要素と固定キャラ依存の要素を分けて記録してください。
```
