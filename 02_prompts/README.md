## ■ プロンプトビューアー向けの書き方

`プロンプトビューアー.html` で一括コピーしやすくするため、今後のプロンプトは以下の形を基本にします。

### 1ファイルの基本構成

```md
# タイトル

## 目的

このプロンプトで作るもの、使う場面、注意点を書く。

## 日本語プロンプト

生成にそのまま貼れる本文を書く。

## English Prompt

英語で生成にそのまま貼れる本文を書く。

## ネガティブプロンプト

避けたい要素をまとめる。

## 補助ブロック

必要な場合だけ、背景、切り抜き、三方向、マネキン、エフェクトなどの追加指定を書く。
```

### 書き方ルール

- コピー対象の本文は、説明文と混ぜない。
- 生成に貼る本文は、`日本語プロンプト`、`English Prompt`、`ネガティブプロンプト` の見出し下へまとめる。
- 1ブロックは「一度にコピーして使える単位」にする。
- メモ、判断、理由、TODOは、`目的` や `運用メモ` に分ける。
- 参照画像のURLや権利メモは、生成本文ではなく資料メモ側に置く。
- Pinterestなど外部資料は、画像そのものを再現させる文ではなく、観察した条件を文章化して使う。
- 三方向GIF用のプロンプトでは、`正面`、`左側面`、`右側面`、`01構え`、`02途中`、`03到達` の語を固定して使う。
- 背景指定、切り抜き指定、マネキン構図指定は共通ブロック化し、毎回本文へ手で書き直さない。
- GIF作成手順、保存方針、チェックリスト、修正運用、出力後の作業メモは、生成用プロンプト本文に混ぜない。
- 動作別プロンプトは、`共通プロンプト` と `01 前手直突き` のような個別ブロックに分ける。ビューアーでは「共通+コピー」で、共通条件と個別動作だけをまとめてコピーする。

### 一括コピー時の前提

- `生成用セットをコピー`: プロンプト、ネガティブ、生成に使える補助ブロックだけをコピーする。GIF手順や保存方針などの運用メモは除外する。
- `共通+コピー`: 共通プロンプトと、選んだ個別ブロックだけをコピーする。基礎動作ごとの生成で使う。
- `プロンプト本文だけコピー`: `prompt` ブロックのみをまとめてコピーする。
- `全ブロックをコピー`: 資料化や確認用に、そのページの全ブロックをコピーする。生成用ではなく確認用。
