# マネキン経由 戦闘イラスト制作ワークフロー

## 目的

ChatGPTの画像生成で、キャラクターの外見を維持しながら複雑な戦闘構図を作るための手順。

ポーズ、キャラクターデザイン、衣装、エフェクトを一度に決めようとすると、関節、顔の向き、指、衣装丈などが崩れやすい。そのため、以下の役割へ分離する。

1. 技の目的と身体運動を文章で設計する。
2. 白い3Dデッサン人形でポーズを可視化する。
3. 立ち絵とマネキン資料を同時に参照し、キャラクター化する。
4. 技ごとに検品し、問題のある画像だけを再生成する。

この方式では、マネキン画像を完成品ではなく「骨格、重心、カメラ、顔向きの指示書」として扱う。

## 基本原則

### 参考画像の役割を分ける

- **立ち絵**: 顔、髪型、体格、衣装、装飾、靴、配色の正本。
- **マネキン**: 姿勢、支持脚、重心、関節角度、手足の前後関係、カメラ方向の正本。
- **文章プロンプト**: 技の意味、視線、衣装の動き、エフェクト、禁止事項を補う。

プロンプト内で、どの画像が何の正本かを明記する。

### 構図検討と完成生成を分ける

- マネキン段階は複数ポーズを一枚へまとめてよい。
- キャラクター化の段階は、一枚につき一人、一技だけ生成する。
- 「複数種類を作成」は、分割シートではなく独立画像を順番に生成すると明記する。

### マネキンを絶対視しない

マネキンの関節や手の形にも誤りは発生する。文章側に次の条件を残す。

```text
指定ポーズの支持脚、重心、身体の向き、手足の前後関係を維持する。
人体として不自然な関節角度だけを、技の目的を変えずに修正する。
```

## 制作フロー

## 1. 技を文章で設計する

各技について、最低限以下を決める。

```text
技名:
技の目的:
動作の瞬間:
支持脚:
重心:
胸郭と骨盤の向き:
両手の役割:
顔と視線:
カメラ方向:
遠近:
静と動:
エフェクトの開始点と出口:
重点確認する関節:
```

技名だけでは、生成モデルが一般的な攻撃姿勢へ置き換えやすい。「何をしている技か」と「どの瞬間か」を必ず記述する。

例:

```text
技名: 光華拳・返葉
技の目的: 後退しながら力を引き込み、側方へ反転させる
動作の瞬間: 重心が後脚へ移り、前方の力をS字に逃がす途中
支持脚: 後脚
顔と視線: 後退方向ではなく前方の相手を見る
```

## 2. マネキン一覧を生成する

複数技を比較するため、白い3Dデッサン人形は一枚の一覧画像へまとめる。

```text
白い3Dデッサン人形を使い、指定した技を一枚のポーズ資料へまとめる。
各パネルに一体、一技だけを配置する。
全パネルで同じ頭身、体格、材質、関節構造を維持する。
髪、衣装、装飾、武器は付けない。
薄いグレー背景、薄い床面、接地影を使う。
各人物の全身、両手、両足をパネル内へ収める。
```

### マネキン段階の確認項目

- 技の意図がシルエットだけで読めるか。
- 支持脚と重心が成立しているか。
- 胸郭と骨盤が技の方向へつながっているか。
- 顔がカメラではなく技の方向を向いているか。
- 前進、後退、回転の違いが明確か。
- 両手の役割と奥行きが分かるか。
- 関節が人体として可能な角度か。

問題のあるポーズだけを修正する。すべてを作り直す必要はない。

## 3. キャラクター化する

立ち絵とマネキン一覧を同時に添付する。

```text
参考画像1はキャラクター外見の正本。
参考画像2は指定番号のポーズ、支持脚、重心、関節、カメラ方向の正本。
参考画像2の人形の顔、材質、体格は引き継がない。
指定番号以外のポーズを混合しない。
```

### 出力方法

```text
各技を独立した画像として一枚ずつ生成する。
一枚に複数ポーズを並べない。
モデルシート、グリッド、分割パネル、一覧表、コラージュにしない。
一枚につきキャラクター一人、一つの技だけを描く。
```

最も安定する運用は、共通プロンプトを渡した後に番号を一つずつ指定する方法。

```text
まず01だけを、新しい独立した一枚として生成する。
```

次の画像では以下のように番号だけを変更する。

```text
次は02だけを、新しい独立した一枚として生成する。
外見と共通画像仕様は直前と同じ条件を維持する。
```

## リーファンで有効だった固定制御

### 顔と視線

生成モデルは顔をカメラ正面へ戻しやすい。

```text
顔は原則としてカメラではなく、技の到達方向を見る。
鼻先、両目、顎を技の方向へそろえる。
首だけをカメラ正面へ戻さない。
回転技では、顔と視線を身体の回転より少し先行させる。
```

### ショートパンツ

「ショートパンツ」だけでは丈が画像ごとに変化する。比率と構造を指定する。

```text
ドレスの下に不透明な淡い桃色のフィットショートパンツを着用する。
股下は太腿全長の約3分の1。
左右同じ丈にする。
裾は太腿上部を水平に一周する。
動作やカメラ角度によって短縮しない。
ハイウエストで骨盤を完全に覆う。
下着として描かない。
```

### 靴

蹴り足や前景の足は裸足へ変化しやすい。両足を個別に固定する。

```text
支持脚と動作脚の両方に、正本と同じ閉じた桃色の靴を維持する。
足指や生身の足裏を描かない。
踵、靴底、つま先の方向は靴の輪郭で表現する。
```

### 手と指

```text
すべての手は五本指。
四本の指と親指一本を明確に分ける。
掌の向きと親指の位置を一致させる。
左右の親指を鏡像関係にする。
リボン、蔦、光、花弁を指へ重ねない。
```

真正面を向く掌は破綻しやすい。可能なら少し斜めにし、左右の掌へ角度差と奥行き差を付ける。

### 衣装と装飾

- 布袖、分離袖、長袖を追加しない。
- 上腕の蓮華と細いリボンを維持する。
- 前掛けは膝上から膝付近の長さを保つ。
- 布で股関節、膝、支持脚を完全に隠さない。
- 前掛け、左右の裾、インナー、脚の前後関係を明記する。

### エフェクト

エフェクトは技の意味を補助し、身体を置き換えない。

```text
エフェクトは一本の連続した流れに限定する。
開始点と画面から抜ける出口を指定する。
閉じた魔法陣、複数の輪、爆発へ変化させない。
顔、指、手首、膝、足首を隠さない。
```

## 検品順序

一枚ずつ、以下の順序で確認する。

1. 技の目的がポーズから読めるか。
2. 前進、後退、回転、受け流しの方向が正しいか。
3. 支持脚と重心が成立しているか。
4. 顔と視線が技の方向を向いているか。
5. 肩、肘、手首、指が自然につながっているか。
6. 骨盤、膝、足首、靴が自然につながっているか。
7. 指は五本で、親指の位置が正しいか。
8. 両足に指定の靴があるか。
9. インナーの丈と衣装構造が統一されているか。
10. エフェクトが技を補助し、身体を隠していないか。

判定は次の三段階にする。

- **採用**: 技、身体、外見が成立している。
- **条件付き採用**: 小さな視線やエフェクト位置の問題だけ。
- **再生成**: 技そのものが別の動作に見える、重心が破綻している。

## 今回確認できた失敗パターン

### 技が一般的な動作へ置き換わる

- 穿掌が下段への払いになる。
- 後退技が前方への掌打になる。
- 横蹴が舞踊的な開脚になる。

対策は、技名ではなく「支持脚」「重心移動」「接触部位」「動作の瞬間」を書くこと。

### 左右対称へ寄りすぎる

双掌技は両手が同じ形、同じ距離、同じ角度になりやすい。

```text
一方の掌を少し前景へ、もう一方を少し奥へ置く。
両掌の角度と大きさに自然な遠近差を付ける。
```

### 顔がカメラ目線へ戻る

「集中した表情」だけでは正面顔になりやすい。鼻先と視線の到達方向まで指定する。

### 衣装寸法が画像ごとに変わる

「長め」「短すぎない」ではなく、太腿に対する比率、裾位置、左右同寸を指定する。

### 一括生成が一枚の一覧画像になる

完成画像では「独立した画像として一枚ずつ」「グリッド禁止」を明記し、番号ごとに生成を実行する。

## 今回の結果

独自技5種の生成では、次の傾向となった。

- **光華拳・環流**: ポーズ、視線、開いた円弧が明確で採用可能。
- **光華拳・旋脈**: 軸足、顔の先行、上昇螺旋が成立し採用可能。
- **光華拳・蓮衝**: 身体は成立。顔向き、掌の前後差、光点位置に改善余地。
- **光華拳・穿華**: 穿掌より下段払いに見えるため再生成推奨。
- **光華拳・返葉**: 後退と反転が弱く、前方攻撃に見えるため再生成推奨。

外見面では、数値的な丈指定と両足への靴指定により、ショートパンツと靴の統一性が改善した。

## 保存と管理

- 構図検討用マネキンは参考資料として保存する。
- ChatGPTから取得した未整理画像は一時フォルダへ置く。
- 採用候補はキャラクターの `04_selected` へコピーする。
- 再生成時は元画像を上書きせず、技名と版番号を付ける。
- 採用理由と残存する問題を `採用メモ.md` に記録する。

推奨ファイル名:

```text
リーファン_光華拳_穿華_v01.png
リーファン_光華拳_穿華_v02.png
リーファン_光華拳_環流_採用.png
```
